# Pitch Pattern Distiller

## Quick Start

Convert a source speech into reusable patterns without copying the source. Treat the source as evidence for structure, not as text to reuse.

## Distillation Workflow

1. **Capture the source boundary**
   - Source type: transcript, captions, notes, public speech, ad, livestream, or video
   - Reliability: verbatim, approximate, machine-generated, or reconstructed
   - Rights boundary: what can be quoted, paraphrased, or only analyzed

2. **Segment by persuasion move**
   - Opening hook
   - Buyer pain or desire
   - Credibility proof
   - Objection handling
   - Contrast or value reframing
   - Emotional scene
   - Call to action

3. **Name the pattern**
   - Use concrete names like "proof ladder", "honest friction", or "occasion reframing".
   - Avoid generic labels like "good hook" or "emotional marketing".
   - The name must explain how to reuse the move.

4. **Write a pattern card**
   - Pattern name
   - Source move in one sentence
   - Formula
   - Best-fit products
   - Failure modes
   - Evidence needed before publishing
   - One neutral example
   - One applied use-case example

5. **Verify transfer**
   - Apply the pattern to a different product than the source.
   - Check whether the adapted pitch still works without borrowed celebrity authority.
   - Mark claims needing product evidence.

## Pattern Card Template

```markdown
## [Pattern Name]

Source boundary:
Source move:
Reusable formula:
Best for:
Avoid when:
Evidence needed:

Neutral example:
Applied use case:
Verification test:
```

## Quality Bar

- Good pattern cards are product-agnostic but not vague.
- Do not preserve source wording unless the user asks for short compliant quotes and rights allow it.
- Do not turn every moment into a pattern. Keep only moves that transfer to another product.
- For celebrity examples, separate "public persona effect" from "endorsement claim".
