# Evidence Supplement

Use this guide when a pitch style needs proof the user has not supplied. The goal is to make the pitch more convincing without inventing claims.

## Evidence Priority

1. **User-supplied source files**: product specs, price sheet, test report, landing page copy, customer research, screenshots.
2. **Official sources**: brand site, product page, certification body, standards body, government or regulator, platform help center.
3. **Primary observations**: demo footage, visible product dimensions, packaging label, opened product footage, before/after screen capture.
4. **Credible secondary sources**: industry reports, academic papers, reputable media, marketplace listings used only for comparison.
5. **Do not use as proof**: unsourced social posts, anonymous comments, competitor claims without context, AI-generated summaries without source links.

## Search Triggers

Search or ask for evidence before writing when the selected style needs:

- Numbers: price, capacity, duration, conversion rate, weight, size, serving count, battery life, speed, rating, benchmark.
- Metrics: certification result, lab result, test standard, customer count, retention, save/share rate, defect rate.
- Facts: origin, material, variety, process, guarantee, compatibility, source date, delivery limits.
- Comparisons: competitor price, category baseline, before/after result, alternative cost.
- Constraints: who should not buy, failure case, safety boundary, warranty, usage conditions.

## Style-Specific Guidance

- `proof-ladder`: require measurable proof or visible evidence. If there are no numbers, search or ask before writing a strong claim.
- `honest-friction`: require real limits, costs, tradeoffs, return rules, usage constraints, or failure modes.
- `occasion-spark`: require lighter evidence such as serving size, occasion fit, audience context, portability, storage, or timing.

## Claim Handling

- Use a claim in the script only when it is supported by supplied evidence or a cited source.
- If the fact is plausible but unsupported, put it under `Evidence still missing`.
- If the claim is risky, regulated, medical, legal, financial, health-related, or endorsement-based, put it under `Claims not safe to publish yet`.
- Preserve dates for time-sensitive facts such as price, availability, rankings, platform metrics, and regulations.

## Output Ledger

Use this compact evidence ledger in the final answer:

```markdown
Evidence used:
- [Fact] - [source type or link], [date if known]

Evidence still missing:
- [Claim needed for stronger pitch]

Claims not safe to publish yet:
- [Claim that should not appear until verified]
```
